지금 팀원과 내가 각각 자신의 브랜치에 작성한 코드를 푸시했어. 
이제 내가 dev 브랜치에 머지하고싶은데 어떻게 해야해?

각자의 브랜치에 코드를 무사히 푸시했다니 프로젝트 협업이 아주 순조롭게 진행되고 있네!
이제 작성한 코드들을 dev 브랜치로 합칠(Merge) 차례야. 깃허브에서 여러 명의 팀원과 협업할 때는 로컬 터미널에서 직접 머지하는 것보다 깃허브 웹사이트에서 Pull Request(PR)를 생성해서 머지하는 방식을 강력히 추천해. 이 방식이 코드 충돌(Conflict)을 시각적으로 확인하기 쉽고, 나중에 누가 언제 어떤 코드를 합쳤는지 기록이 명확하게 남아서 훨씬 안전하거든.
두 가지 방법 모두 알려줄 테니 편한 방법을 선택해 봐.
방법 1: 깃허브 웹사이트에서 Pull Request로 머지하기 (가장 추천)
깃허브 리포지토리 접속: 프로젝트 깃허브 웹 페이지로 들어가.
Compare & pull request 클릭: 방금 브랜치에 푸시를 했기 때문에 화면 상단에 노란색 바탕으로 Compare & pull request 버튼이 떠 있을 거야. 그 버튼을 눌러줘. (안 보인다면 탭에서 'Pull requests'를 누르고 'New pull request'를 클릭해)
브랜치 방향 설정: 머지가 일어나는 방향을 맞춰주는 게 제일 중요해.
base: dev (코드가 모일 목적지)
compare: 본인의 브랜치 이름 (내가 작업해서 합칠 코드)
이렇게 compare에서 base로 코드가 들어가는 방향이 맞는지 꼭 확인해.
PR 생성: 코드가 합쳐졌을 때 문제가 없는지(Able to merge) 확인되면, 제목과 간단한 작업 내용을 적고 Create pull request 버튼을 눌러.
머지 완료: 페이지가 넘어가고 코드 충돌이 없다면 초록색 Merge pull request 버튼이 활성화될 거야. 클릭하고 Confirm merge를 누르면 깃허브 상의 dev 브랜치에 내 코드가 성공적으로 합쳐져.
팀원의 브랜치도 똑같은 과정으로 PR을 만들어서 dev에 합치면 돼.
